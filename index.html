Autenticación y Sesión (GitHub CLI)

Guía de Instalación de GitHub CLI

Instalación mediante PowerShell

Para instalar GitHub CLI en Windows usando PowerShell, sigue estos pasos:

1
Instalar Winget (si no lo tienes)

Winget viene preinstalado en Windows 11. Para Windows 10, descárgalo desde Microsoft Store buscando "App Installer"

2
Instalar GitHub CLI con Winget
winget install --id GitHub.cli

Este comando instalará la versión más reciente de GitHub CLI

3
Verificar la instalación
gh --version

Deberías ver la versión instalada de GitHub CLI

Nota importante: Después de instalar GitHub CLI, es posible que necesites reiniciar tu terminal o PowerShell para que los comandos estén disponibles.

Comandos de Autenticación

gh auth login
Inicia sesión en GitHub a través de la línea de comandos. Te guía paso a paso para autenticarte con tu cuenta de GitHub.
Te pedirá elegir entre GitHub.com o GitHub Enterprise, y el método de autenticación (navegador o token)
gh auth status
Muestra el estado de autenticación actual, incluyendo el usuario conectado y los scopes disponibles.
gh auth logout
Cierra la sesión de GitHub CLI y elimina las credenciales almacenadas localmente.
gh auth refresh
Actualiza y expande los permisos de autenticación. Útil cuando necesitas agregar más scopes a tu sesión.
gh auth setup-git
Configura Git para usar GitHub CLI como ayudante de credenciales para operaciones HTTPS.
gh repo create nombre-repo --public
Crea un nuevo repositorio público en GitHub desde la línea de comandos. Puedes usar --private para crear un repositorio privado.
Ejemplo: gh repo create mi-proyecto --public --description "Mi nuevo proyecto"

Configuración Inicial

git config --global user.name "Tu Nombre"
Configura tu nombre de usuario para todos los repositorios. Este nombre aparecerá en los commits.
Ejemplo: git config --global user.name "Juan Pérez"
git config --global user.email "email@ejemplo.com"
Configura tu email para todos los repositorios. Es necesario para identificar al autor de los commits.
Ejemplo: git config --global user.email "juan@email.com"
git config --list
Muestra todas las configuraciones actuales de Git (usuario, email, etc.).

Crear y Clonar Repositorios

git init
Inicializa un nuevo repositorio Git en el directorio actual. Crea la carpeta oculta .git donde se guarda todo el historial.
Úsalo cuando empiezas un proyecto nuevo desde cero
git clone [url]
Crea una copia local de un repositorio remoto. Descarga todo el historial y archivos del proyecto.
Ejemplo: git clone https://github.com/usuario/proyecto.git

Realizar Cambios

git status
Muestra el estado actual del repositorio: archivos modificados, archivos en staging, branch actual, etc. Es uno de los comandos más usados.
git add [archivo]
Añade archivos al área de staging (preparación). Los archivos deben estar en staging antes de hacer commit.
git add index.html → añade un archivo específico
git add . → añade todos los archivos modificados
git commit -m "mensaje"
Guarda los cambios del área de staging en el historial del repositorio. El mensaje debe describir qué cambios se hicieron.
Ejemplo: git commit -m "Agrega página de contacto"
git commit -am "mensaje"
Hace add y commit en un solo paso para archivos que ya están siendo rastreados (tracked).
git diff
Muestra las diferencias entre los archivos modificados y la última versión guardada.

Ver Historial

git log
Muestra el historial de commits del repositorio con detalles completos (autor, fecha, mensaje).
git log --oneline
Muestra el historial de commits de forma compacta (una línea por commit). Útil para ver el historial rápidamente.
git log --graph
Muestra el historial con un gráfico ASCII de las ramas y merges. Ayuda a visualizar la estructura del proyecto.

Trabajar con Ramas (Branches)

git branch
Lista todas las ramas locales. La rama actual se marca con un asterisco (*).
git branch [nombre]
Crea una nueva rama con el nombre especificado. Las ramas permiten trabajar en características separadas.
Ejemplo: git branch desarrollo
git checkout [rama]
Cambia a la rama especificada. Actualiza los archivos del directorio de trabajo a esa rama.
Ejemplo: git checkout desarrollo
git checkout -b [nombre]
Crea una nueva rama y cambia a ella inmediatamente (combina branch + checkout).
Ejemplo: git checkout -b nueva-funcionalidad
git merge [rama]
Fusiona la rama especificada con la rama actual. Combina los cambios de ambas ramas.
Ejemplo: git merge desarrollo (estando en main)
git branch -d [rama]
Elimina una rama local. Solo funciona si la rama ya fue fusionada (merge).
git branch -D [rama]
Elimina una rama local forzadamente, incluso si no ha sido fusionada.

Repositorios Remotos

git remote add origin [url]
Conecta tu repositorio local con un repositorio remoto (GitHub, GitLab, etc.). "origin" es el nombre estándar.
Ejemplo: git remote add origin https://github.com/usuario/proyecto.git
git remote -v
Muestra los repositorios remotos configurados con sus URLs.
git push origin [rama]
Envía los commits de tu rama local al repositorio remoto. "Sube" tus cambios a GitHub/GitLab.
Ejemplo: git push origin main
git push -u origin [rama]
Primera vez que subes una rama. El -u guarda la configuración para futuros push.
git pull
Descarga los cambios del repositorio remoto y los fusiona con tu rama actual. Es fetch + merge.
git fetch
Descarga los cambios del repositorio remoto pero NO los fusiona. Te permite revisar antes de integrar.

Deshacer Cambios

git restore [archivo]
Descarta los cambios de un archivo modificado y lo restaura a la última versión del commit.
Ejemplo: git restore index.html
git restore --staged [archivo]
Quita un archivo del área de staging (deshace git add) pero mantiene los cambios en el archivo.
git reset [archivo]
Quita archivos del área de staging. Alternativa a git restore --staged.
git reset --hard [commit]
Vuelve a un commit específico eliminando todos los cambios posteriores. ⚠️ CUIDADO: es irreversible.
git revert [commit]
Crea un nuevo commit que deshace los cambios de un commit anterior. Forma segura de deshacer.

Otros Comandos Útiles

git stash
Guarda temporalmente los cambios sin hacer commit. Útil cuando necesitas cambiar de rama pero no quieres hacer commit.
git stash pop
Restaura los cambios guardados con git stash y los elimina del stash.
git rm [archivo]
Elimina un archivo del repositorio y del directorio de trabajo.
git mv [origen] [destino]
Mueve o renombra un archivo y prepara el cambio para commit.
git tag [nombre]
Crea una etiqueta (tag) para marcar un punto específico del historial (versiones, releases).
Ejemplo: git tag v1.0.0
git --help
Muestra la ayuda general de Git. También puedes usar git [comando] --help para ayuda específica.

Flujo de Trabajo Básico

1. git status → Ver qué archivos cambiaron
2. git add . → Preparar todos los cambios
3. git commit -m "descripción" → Guardar los cambios
4. git push origin "rama" → Subir al repositorio remoto
